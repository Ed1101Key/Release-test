# Тестовое задание build/release-инженер ispmanager

С доступом к внешнему GitLab случилась накладка, поэтому передаём тестовое по-старинке.

Задание расчитано на 2 рабочих часа.

## Цель задания

1. Перевести сборку проекта на deb- и rpm-пакеты
2. Обеспечить корректную установку и запуск проекта
3. Оптимизировать Makefile

## Требования

### 1 Требования к Makefile

Реализовать:
* clean - таргет для очистки временных файлов и артефактов
* pkg - таргет для сборки пакета.
* Перенести компиляцию и линковку в процесс сборки пакетов
* Версия пакета определяется автоматически через тег git. При отсутствии тега - версия берётся из переменной VERSION в Makefile.
* Для сборки пакета требуется только выполнить команду `make pkg`
* Тип пакета определяется автоматически в зависимости от ОС, на которой произведён запуск сборки
* Добавить строгие флаги компиляции: `-Wall -Wextra -Werror`
* Артефакты помещаются директорию `dist/`

### 2 Требования к пакетам

Реализовать:
* Корректную сборку пакетов - отсутствие ошибок сборки, линковки, компиляции.
* При сборке пакета не должно появляться `WARNING`. Если их не избежать в отведённое на задание время - описать почему они возникают.
* Установку с автозапуском сервиса
* Корректное обновление пакета на новую версию - (Из-за ограничений кода, остановка процесса происходит в течение 5 секунд после сигнала)
* Корректное удаление сервиса

### 3 Требования к сборочному окружению

* Обеспечить установку необходимых зависимостей - библиотеки `fmt`
* Обеспечить сборку в среде rhel9, rhel10 (вместо rhel можно использовать almalinux)
* Обеспечить сборку в среде Debian 12
* Обеспечить сборку в среде Ubuntu 22

### 3 Ограничения

! Нельзя:

* Изменять файлы в директории src
* Изменять README.md

Приветствуется:
* Автоматизация через GitLab CI (или любую другую платформу) - будет плюсом, но не является требованием
* Улучшения и оптимизации, выходящие за рамки требуемых - если реализация займёт времени больше, чем выделено на задание, то можно описать идеи текстом.

Обязательно:
* Загрузить код в git-репозиторий (GitHub, GitLab, др.)

## Приём задания

Сдачей задания является отправка ссылки на git-репозиторий с выполненным заданием. Проверка и обратная связь в течение 8 рабочих часов.
